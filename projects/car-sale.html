Projects / Case study

Car Sale Management System

Solo • 2024 (C++) → 2025 (Python) PythonPySide6SQLiteOOP

Problem

I wanted to build a system to manage car sales effectively—display listings and add/update/delete cars. The first version was a C++ console app. I then upgraded to Python with a full GUI to simulate a real marketplace.

Approach

  • Structured OOP model: Car, Vehicle, Salesperson, Startup, ManagingSystem, Main, GUI, and a SQL-backed Data layer.
  • Core features: add/search cars, dashboards, export to Excel.
  • Advanced UX: analytics and theme toggling.
  • Migration: C++ console → clean Python OOP architecture.
  • GUI: built with PySide6 for a marketplace-like experience.
  • Persistence: SQLite for listings and sales data.

Results / Impact

  • Moved from console to a user-friendly GUI workflow.
  • Faster decisions via integrated analytics/dashboards.
  • Reduced manual work with Excel exports and persistent storage.

Stack

Python (3.x), PySide6, SQLite, openpyxl
